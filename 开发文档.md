### 前言
+ 这只是一个用来总结官方Api接口参数的文档......
### 本地Api
+ 接口
  ```
  /api       # 对应官方第二个
  /snapi     # 对应官方第四个
  /sapi      # 对应官方第五个（暂未分析出用途）
  ```
+ 数据保存
  ```
  # old（待改善）
  /static/api/{请求类型}/{ID}.js

  # new（待编写）
  /static/api/{请求类型}/load.php        # 配置
  /static/api/{请求类型}/testindex.ht    # 模板
  /static/api/{请求类型}/{ID}/{数据}.js   # 数据
  /static/api/img/{添加顺序}.jfif(png)   # 图片
  /static/api/vod/{腾讯视频ID}.mp4       # 视频
  ```
+ 参数
  ```
  # /api (获取)
  p0         # 游戏ID，见index.js:21
  p1         # 检索类型，如新闻、视频
  page       # 翻页用的，见新闻页
  pagesize   # 返回数据数量，最多30
  order      # ???
  r0         # 返回类型，如script代表js
  r1         # Status，需在后端替换数据
  type       # ???
  id         # 大概率是唯一识别码
  source     # 客户端类型，Web_pc

  # /api (上报)
  p0         # 游戏ID
  p1         # updateTotalPlay
  p2         # 新闻或视频的ID
  p3         # ???
  p5         # ???
  啥也不用管，返回空白即可

  # /snapi (新闻)
  p0         # 游戏ID
  source     # 客户端
  id         # 一定是唯一识别码

  # /sapi (待分析)
  ```
+ 逻辑（思路）
  ```
  不能只保存返回数据，会有很多重复，要分析结构，动态拼接
  使用之前拼接html的方法将数据按照请求参数进行拼接
  实现真正的后端
  ```
+ 返回
  ```
  # /api（list页）
  一页最多返回30条数据
  需要替换Status内容
  page       # 当前页
  pagesize   # 当前条目数量
  result     # 条目内容
  total      # 总条目数量
  totalpage  # 以当前条目数量可翻页数
  (totalpage = total / pagesize ?)

  # /api（videos页）
  与上面的基本相同

  # /snapi（news页）
  123
  ```

  ```

  ```
